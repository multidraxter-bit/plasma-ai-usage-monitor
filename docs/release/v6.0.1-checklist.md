# v6.0.1 Release Checklist

This document tracks the stabilization and release procedure for `v6.0.1` ("Ground Truth").

## Pre-flight Checks
- [ ] Ensure `clean-local` removes stale installs:
  ```bash
  just clean-local
  ```
- [ ] Validate versions, hardcoded references, and QML type registrations:
  ```bash
  just check
  ```
- [ ] Run all unit tests:
  ```bash
  just test
  ```
- [ ] Package the source tarball:
  ```bash
  just package
  ```
- [ ] Smoke-check active development install:
  ```bash
  just smoke
  ```
- [ ] Perform strict release-gate checks (requires `rpmlint` and `appstream` installed on Fedora):
  ```bash
  just release-check
  ```

## Local Validation
- [ ] Run `just install` to deploy locally.
- [ ] Reload plasmashell with `just reload`.
- [ ] Open the widget and verify functionality.
- [ ] Check the widget's "About" or "AppInfo" page clearly shows `v6.0.1`.

## Packaging
- [ ] Generate an SRPM/RPM for Fedora/COPR using `just copr-srpm`.
- [ ] Ensure COPR build completes without errors.

## Tagging & Publishing
- [ ] Create a git tag for `v6.0.1`:
  ```bash
  git tag v6.0.1
  git push origin v6.0.1
  ```
- [ ] Draft a new GitHub Release against `v6.0.1`.
- [ ] Attach `plasma-ai-usage-monitor-6.0.1.tar.gz` from the `dist/` directory to the GitHub release.
- [ ] Verify the GitHub repository tags and README clearly reflect `v6.0.1` as the active version.
