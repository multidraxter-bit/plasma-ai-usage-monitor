# v7.0.0 Beacon Release Checklist

This checklist tracks the Fedora KDE 44 native reliability release.

## Scope

- Fedora KDE 44 / Plasma 6.6 is the required release validation target.
- Older Fedora releases are legacy compatibility only and are not required CI targets.
- The widget remains KDE Plasma native, local-first, and desktop-focused.
- Backend services, web dashboards, PostgreSQL, account systems, and multi-user sync are out of scope.

## Required Validation

Run these before tagging:

```bash
just check
just release-check
just fedora44-check
cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug -DBUILD_TESTING=ON
cmake --build build --parallel
ctest --test-dir build --output-on-failure
appstreamcli validate com.github.loofi.aiusagemonitor.metainfo.xml
rpmlint plasma-ai-usage-monitor.spec
bash scripts/package_source_tarball.sh --check
bash scripts/package_plasmoid.sh --check
python3 scripts/check_package_payload.py
```

## Release Artifacts

- `plasma-ai-usage-monitor-7.0.0.tar.gz`
- `com.github.loofi.aiusagemonitor.plasmoid`
- Git tag `v7.0.0`
- GitHub Release titled `v7.0.0 Beacon`

## Manual Plasma QA

- Validate in a real Fedora KDE 44 Plasma session, not only a container.
- Confirm Trust Center labels are clear and do not imply exact billing when data is estimated or self-tracked.
- Confirm Browser Sync Labs never logs cookies, cookie headers, tokens, or secrets.
- Confirm Simple / Operator / Analyst modes and Only Problems filtering work with enabled providers and local tools.
